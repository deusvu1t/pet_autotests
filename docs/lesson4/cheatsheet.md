# Шпаргалка — урок 4 (XPath)

## Базовый синтаксис

| Цель                          | XPath                              |
|-------------------------------|-------------------------------------|
| Тег                           | `//button`                          |
| Любой элемент                 | `//*`                               |
| По id                         | `//*[@id='login']`                  |
| По атрибуту                   | `//input[@name='q']`                |
| Несколько атрибутов           | `//input[@type='submit'][@value='Search']` |
| Тег + класс (через contains)  | `//button[contains(@class,'btn')]`  |

> ⚠ `/` в начале = absolute (плохо). `//` в начале = relative (нормально).

---

## Атрибутные функции

| Функция                   | Что делает                       | Пример                                |
|---------------------------|----------------------------------|---------------------------------------|
| `contains(@a, 'v')`       | подстрока                        | `//*[contains(@class,'btn')]`         |
| `starts-with(@a, 'v')`    | начало строки                    | `//*[starts-with(@id,'user')]`        |
| `not(@a)`                 | без атрибута                     | `//input[not(@hidden)]`               |
| `not(@a='v')`             | не такое значение                | `//a[not(@target='_blank')]`          |

> XPath 1.0 (что есть в браузерах) **не умеет** `ends-with()`. Для "заканчивается на" — CSS `[attr$='val']`.

---

## Текстовые функции

| Функция                          | Пример                                 |
|----------------------------------|----------------------------------------|
| `text()='X'`                     | `//button[text()='Login']`             |
| `contains(text(), 'X')`          | `//h2[contains(text(),'Fiction')]`     |
| `normalize-space(text())='X'`    | `//button[normalize-space(text())='Login']` |
| `normalize-space()='X'`          | то же, короче                          |

> `text()` — **прямой** текст узла. Если текст внутри `<span>`, используй `.` или `contains(., 'X')`.

---

## Позиция

| Селектор                | Что выбирает                          |
|-------------------------|----------------------------------------|
| `//li[1]`               | каждый `<li>`, который первый среди братьев |
| `(//li)[1]`             | первый `<li>` во всём документе       |
| `//li[last()]`          | последний среди братьев                |
| `//li[position()=2]`    | то же, что `[2]`                       |
| `//li[position()>1]`    | все, кроме первого                     |

> **Запомнить:** `//tag[N]` ≠ `(//tag)[N]`.

---

## Оси (axes) — главная фишка XPath

| Ось                   | Краткая форма | Что выбирает                            |
|-----------------------|---------------|------------------------------------------|
| `ancestor::`          | —             | Все предки до корня                      |
| `parent::`            | `/..`         | Прямой родитель                          |
| `descendant::`        | `//`          | Все потомки                              |
| `child::`             | `/`           | Прямые потомки                           |
| `following-sibling::` | —             | Соседи после на том же уровне            |
| `preceding-sibling::` | —             | Соседи до на том же уровне               |
| `following::`         | —             | Все элементы после в порядке документа   |
| `preceding::`         | —             | Все элементы до в порядке документа      |
| `self::`              | —             | Текущий узел                             |

```python
# найти карточку по имени и подняться к её корню
"//span[@data-test='product-name' and text()='Pliers']/ancestor::a[starts-with(@data-test,'product-')]"

# найти input, который сразу после label 'Email'
"//label[text()='Email']/following-sibling::input[1]"

# подняться к родителю
"//input[@id='email']/..        # parent
//input[@id='email']/parent::div  # parent с фильтром"
```

## Логика

```xpath
//tag[@a='1' and @b='2']
//tag[@a='1' or @b='2']
//tag[not(@hidden)]
//tag[contains(@class,'x') and starts-with(@id,'y')]
```

## XPath ↔ CSS — соответствия

| Задача                          | CSS                          | XPath                                        |
|---------------------------------|------------------------------|----------------------------------------------|
| По тегу                          | `button`                     | `//button`                                   |
| По id                            | `#x`                         | `//*[@id='x']`                               |
| По атрибуту                      | `[name='q']`                 | `//*[@name='q']`                             |
| Начинается с                     | `[id^='u']`                  | `//*[starts-with(@id,'u')]`                  |
| Содержит                          | `[c*='p']`                   | `//*[contains(@c,'p')]`                      |
| Прямой ребёнок                   | `ul > li`                    | `//ul/li`                                    |
| Любой потомок                    | `ul li`                      | `//ul//li`                                   |
| Сосед сразу после                | `h2 + p`                     | `//h2/following-sibling::p[1]`               |
| **Родитель**                     | —                            | `//*[@id='x']/..`                            |
| **Предок**                       | —                            | `//*[@id='x']/ancestor::section`             |
| **Сосед до**                     | —                            | `//x/preceding-sibling::label`               |
| По тексту                         | `:has-text('X')` (PW)        | `//*[text()='X']`                            |

## XPath ↔ Playwright API — что когда

| Задача                            | Playwright API                          | XPath                                    |
|-----------------------------------|------------------------------------------|------------------------------------------|
| По роли                            | `get_by_role("button", name="X")`        | `//button[text()='X']`                   |
| По label                           | `get_by_label("Email")`                  | `//label[text()='Email']/following::input[1]` |
| По тексту                          | `get_by_text("X")`                       | `//*[text()='X']`                        |
| N-й элемент                        | `loc.nth(N)`                             | `(//tag)[N+1]`                           |
| Фильтр по тексту                   | `loc.filter(has_text="X")`               | `//*[.//*[contains(.,'X')]]`             |
| Фильтр по потомку                  | `loc.filter(has=...)`                    | `//x[.//y]`                              |
| Родитель                            | — (нет прямого)                          | `/..` или `ancestor::`                   |

**Правило:** XPath берётся только тогда, когда задача — родители/предки/нормализация пробелов.

## Антипаттерны

| ❌                                              | ✅                                              |
|------------------------------------------------|------------------------------------------------|
| `/html/body/div/...`                           | `//tag[@attr='value']`                         |
| `(//button)[5]` (магический индекс)            | `//button[@data-test='submit']`                |
| `//*[@id='ng-1234567']` (сгенерированный id)   | `//*[@data-test='...']` или `get_by_role`      |
| `loc.text_content()` + `assert`                | `expect(loc).to_have_text(...)`                |
| `page.wait_for_timeout(3000)`                  | ничего — `expect` ждёт сам                     |
| `//button[text()='Login']`                     | `get_by_role("button", name="Login")`          |

## Быстрая памятка

1. **Только `//`** в начале, никогда `/`.
2. **`text()`** — прямой текст. Для вложенного — `.` или `contains(., 'X')`.
3. **`normalize-space()`** — если в HTML могут быть пробелы.
4. **`//tag[1]` ≠ `(//tag)[1]`**.
5. **Ось** = `ось::фильтр`. `ancestor::`, `parent::`, `following-sibling::` — главные.
6. **XPath — последний выбор.** `get_by_*` → CSS → XPath.
